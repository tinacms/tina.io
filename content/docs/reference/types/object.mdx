The `object` field is a dynamic field type that creates a **nested page** in the CMS with the data it contains.

This can be used to **create repeated data structures** or **group related data** in the CMS.

## Type Definition

```ts
type ObjectField = {
  name: string,
  type: 'object',
  label?: string,
  description?: string,
  list?: boolean,
  fields?: Field[]
  templates?: Template[]
  templatesKey?: string
  ui?: {
    visualSelector?: boolean,
    defaultItem?: Record<string, any> | () => Record<string, any>,
    itemProps?: (
      item: Record<string, any>
    ): {
      label?: string
    }
  }
}
```

This is just a reference. The generic [Field](http://localhost:3000/docs/reference/fields) type definition contains further usage information.

One **one** of `fields` or `templates` should be defined.

### Conceptualising the object type

Importantly, there are two key ways to use the object type...

1. Using `fields`, the object wraps its inner fields in a sub-window.\
   \
   With `list` set to `true`, the object repeats inner fields in a sub-window for each list item.![](</img/docs/reference/Screen Recording 2025-03-14 at 11.38.07 am.gif>)
2. With `list` set to `true`, and using `templates` (as opposed to `fields`). \
   \
   The user can select between associated [templates](/docs/reference/templates) – this appears as above, but pressing the `+`  gives you the option to choose one of a set of options\
   ![](</img/docs/reference/Screen Recording 2025-03-14 at 12.03.21 pm.gif>)

## The Visual Selector

This visual block selector allows editors to select Templates with images as well as text.

To enable the visual block selector, the `visualSelector` property must be true. In the editor, this also opens a new window for template selection.

![](</img/docs/reference/Screen Recording 2025-03-14 at 11.43.03 am.gif>)

To get images for a certain template, you need to provide the image with the template's `ui.previewSrc` property. This is relative to your [public folder](/docs/reference/media/repo-based) defined in the TinaCMS configuration.

```javascript
const showcaseTemplate: Template = {
  label: 'Showcase',
  name: 'showcase',
  ui: {
    previewSrc: '/img/blocks/features.png',
  },
  fields: [
    //...
  ]
};
```

## Custom List Labels

By default, object lists label each element with the Template `label` or the object's `label` + `" Item"`.

![list UI](/img/docs/extending-tina/Extending_Tina_No_List_Props.png)

If you have a number of objects in the list, this can be customised per element with the `itemProps` function property.

* set this on the object list if using `fields`
* set this on each template is using `templates`

![List UI with label prop](/img/docs/extending-tina/Extending_Tina_Label_List_Props.png)

## Custom Template Identifiers

When using the templates property, your document stores the each chosen template under a key, `_template`.

```markdown
---
farm:
  - name: Tina
    _template: llama
  - name: Napoleon
    _template: farmer
---
```

You can customise this with the `templatesKey` property – the given key will be used to define the template rather than the default `_template`.

## Examples

### Simple field

```javascript
{
  label: "Testimonial",
  name: "testimonial",
  type: "object",
  fields: [
    {
      label: "Author",
      name: "author",
      type: "string"
    },
    {
      label: "Quote",
      name: "quote",
      type: "string",
      ui: {
        component: "textarea"
      }
    }
  ]
}
```

### Simple field (templates)

```javascript
{
  label: "Page Blocks",
  name: "pageBlocks",
  type: "object",
  list: true,
  templates: [
    {
      label: "CTA",
      name: "cta",
      fields: [...]
    },
    {
      label: "Testimonial",
      name: "testimonial",
      fields: [...]
    }
  ]
}
```

![](/img/docs/reference/page-blocks.png)

### Simple wrapper field

Number, boolean, datetime, reference and rich-text field types can be used as the sole field of an object to create a list of one of those types.

```javascript
{
  label: "Author List",
  name: "authorList",
  type: "object",
  list: true,      
  fields: [
    {
      label: 'Author',
      name: 'author',
      type: 'reference',
      collections: ['author'],
    },
  ]
}
```

### Nested object fields

This example uses both templates and fields.

* The `featureItem` objects stores a vector array `[headline, description, buttons]`.
* Each feature item in the list has a unique label, so items can be identified in the editor (lines 12-14)
* The buttons element of the `featureBlock` array is also an object list, with the option to choose between 3 templates for each list element.

```javascript
import type { Template, TinaField } from 'tinacms';

export const featuresTemplate: Template = {
  label: 'Feature Block',
  name: 'featureBlock',
  fields: [
    {
      name: 'featureItem',
      label: 'Feature Items',
      type: 'object',
      list: true,
      ui {
      itemProps: (item) => {
        return { label: item?.headline };
      },
    },
    fields: [
      { name: 'headline', label: 'Headline', type: 'string' },
      {
        name: 'description', label: 'Description',
        type: 'string', ui: { component: 'textarea' }
      },
      {
        name: 'buttons',
        label: 'Buttons',
        list: true,
        type: 'object',
        ui: {
          visualSelector: true,
          itemProps: (item) => {
            return { label: item?.label };
          },
        },
        templates: [
          actionsButtonTemplate as Template,
          codeButtonTemplate as Template,
          modalButtonTemplate as Template
        ],
      },
    ] as TinaField[], 
    },
  ],
};
```

### Field with default values

When a new item is added to the list, it will be created with these defaultItem values.

```javascript
{
  label: "Testimonials",
  name: "testimonials",
  type: "object",
  list: true,
  defaultItem: {
    author: "Judith Black",
    role: "CEO",
    quote: "Lorem ipsum dol..."
  }
  fields: [
    {
      name: "author",
      type: "string"
    },
    {
      name: "role",
      type: "string"
    },
    {
      name: "quote",
      type: "string"
      ui: {
        component: "textarea"
      }
    }
  ]
}
```

### Field with custom reference key

Setting templateKey here changes our saved data to the below, where the key defining the template has been set to "type".

```javascript
{
  name: "farm",
  type: "object",
  list: true,
  templateKey: "type",
  templates: [
    {
      name: "llama",
      fields: [
        {
          name: "name",
          type: "string",
        },
      ],
    },
    {
      name: "farmer",
      fields: [
        {
          name: "name",
          type: "string",
        },
      ],
    },
  ],
},
```

```markdown
---
farm:
  - name: Tina
    type: llama
  - name: Napoleon
    type: farmer
---
```
