A dedicated file input that stores the URL or path of an uploaded image.

It supports [external media library integration](http://localhost:3000/docs/reference/media/overview#external-media-providers), but also [repo-based media configurations](http://localhost:3000/docs/reference/media/repo-based).

Images can be uploaded to the image component with drag-and-drop actions, or with the dedicated media manager that comes with the CMS.

![](/img/docs/reference/SCR-20250313-nzft.png)

## Type Definition

```ts
type ImageField = {
  name: string,
  type: 'image',
  label?: string,
  description?: string,
  list?: boolean,
  ui?: {
    component?: FC<any> | string | null
    parse?: (value: string, name: string, field: F) => any
    format?: (value: string, name: string, field: F) => any
    validate?: (
      value: string,
      allValues: any,
      meta: any,
      field: UIField<F, Shape>
    ): string | undefined | void
  }
}
```

This is just a reference. The generic [Field](http://localhost:3000/docs/reference/fields) type definition contains further usage information.

## Example

### Simple field

```ts
{
  type: 'image',
  label: 'Hero image',
  name: 'imgSrc',
}
```

![](/img/docs/reference/SCR-20250313-nzau.png)

### Field with image path formatting

If you need to customize how an image is formatted in a field, you can use the `format` & `parse` options.

```js
{
  type: "image",
  name: "heroImg",
  label: "Hero Image",
  ui: {
    format(value) {
      //add leading slash to value if it doesnt exist
      return value.startsWith("/") ? value : `/${value}`;
    },
    parse(value) {
      //remove leading slash if it exists
      return value.startsWith("/") ? value.slice(1) : value;
    },
  }
}
```
