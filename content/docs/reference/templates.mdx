Templates exist as an alternative to the fields type when multiple variants of the same **collection** or **object**.

> For example in a "page" collection there might be a need for a marketing page template and a content page template, both under the collection "page".

## Type Definition

| Property       | Description                                                                           |
| -------------- | ------------------------------------------------------------------------------------- |
| `name`         | The name of the template.                                                             |
| `label`        | The template name as shown in the editor.                                             |
| `fields`       | An array of [fields](/docs/reference/fields/).                                        |
| ui.itemProps   | Controls how each template appears in the editor.                                     |
| ui.defaultItem | An object or a function that returns the default data when a new document is created. |
| ui.previewSrc  | Sample image for this template.                                                       |

## Examples

### Basic example with two templates

```ts
export default defineConfig({
  //...
  schema: {
    collections: [
      {
        name: 'page',
        label: 'A page of the website',
        path: 'content/pages',
        format: 'mdx',
        templates: [
          {
            name: 'content',
            label: 'Content Page',
            fields: [
              // ... content page fields
            ],
          },
          {
            name: 'marketing',
            label: 'Marketing Page',
            inline: true,
            fields: [
              //... marketing page fields
            ],
          },
        ],
      },
    ],
  },
})
```
