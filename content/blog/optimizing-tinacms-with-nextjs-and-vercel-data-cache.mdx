When using TinaCMS with Next.js and the App Router on Vercel, Tina supports **Vercelâ€™s Data Cache**. This caching mechanism enhances performance by storing API responses, reducing redundant requests, and speeding up page loads. While this offers significant benefits, Vercel caches TinaCloudâ€™s API responses for an extended period by default. If you notice stale content, you can manage revalidation in two ways:

1. **Adjust the Revalidation Time**\
   Modify the `revalidate` setting for route segments to control how often Vercel checks for updates. This enables Incremental Static Regeneration (ISR) to refresh pages with new data. [Learn more](https://nextjs.org/docs/app/api-reference/file-conventions/route-segment-config#revalidate).
2. **Use Query Parameters for Fresh Fetches**\
   Add a `revalidate` parameter to your TinaCMS GraphQL queries, ensuring that Next.js fetch requests are updated at a set interval:\
   `const response = await client.queries.page({ relativePath: 'pages/home.mdx' }, { fetchOptions: { next: { revalidate: 60 } } });`\
   More details on Next.js fetch extensions [here](https://nextjs.org/docs/app/api-reference/functions/fetch).

You can find more information in the [TinaCMS docs](https://tina.io/docs/frameworks/next/overview#updates-not-reflecting-correctly-on-page).
