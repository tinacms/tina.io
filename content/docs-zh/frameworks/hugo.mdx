## 介绍

TinaCMS 可以在本地添加到您的 Hugo 站点。在本文档中，我们将指导您完成本地设置，以及在生产站点上进行编辑。

> 您可以参考我们的 [Tina Hugo Starter](https://github.com/tinacms/tina-hugo-starter) 作为示例。

## 入门

在您的站点目录中，运行：

```bash
 npx @tinacms/cli@latest init
```

这将询问您一些设置问题。

当提示输入“**公共资产目录**”时，输入：**static**。

> 如果您是从 Forestry.io CMS 导入站点，某些模型和配置将自动导入。

## 建模您的内容

要在 Tina 中编辑您的站点内容，您可以在 tina/config.ts 文件中建模您的内容。

在[这里](/docs/schema/)了解更多关于内容建模的信息

## 启动 TinaCMS

您可以通过以下方式启动 TinaCMS：

```bash
npx tinacms dev -c "hugo server -D -p 1313"
```

> `hugo server -D -p 1313` 可以替换为您站点的自定义开发命令。

在 TinaCMS 运行时，导航到 `http://localhost:1313/admin`

> ^ 上述默认端口可能会因您的框架而异。

> 提示：如果运行此命令时出现错误，请参阅[常见错误](/docs/forestry/common-errors)页面。

此时，您应该能够看到 Tina 管理界面，选择一个帖子，保存更改，并看到更改已持久化到您的本地 markdown 文件中。
