`object`字段是一种动态字段类型，它在CMS中创建一个包含其数据的**嵌套页面**。

这可以用于在CMS中**创建重复的数据结构**或**分组相关数据**。

## 类型定义

> 有关所有字段类型的其他常见属性，请查看[字段](/docs/reference/fields)类型定义。

`fields`或`templates`中只能定义**一个**。

### 概念化对象类型

重要的是，有两种关键方式使用对象类型...

1. 使用`fields`，对象将其内部字段包装在一个子窗口中。![](</img/docs/reference/Screen Recording 2025-03-14 at 11.38.07 am.gif>)
2. 使用`templates`时，用户可以在关联的[模板](/docs/reference/templates)之间进行选择。![](</img/docs/reference/Screen Recording 2025-03-14 at 12.03.21 pm.gif>)

如上所示，但按下`+`会让您选择一组选项之一。

## 视觉选择器

这个视觉块选择器允许编辑者选择带有图像和文本的模板。

要启用视觉块选择器，`visualSelector`属性必须为true。在编辑器中，这也会打开一个新的模板选择窗口。

![](</img/docs/reference/Screen Recording 2025-03-14 at 11.43.03 am.gif>)

要为某个模板获取图像，您需要通过模板的`ui.previewSrc`属性提供图像。这是一个URL，或相对于您在TinaCMS配置中定义的[公共文件夹](/docs/reference/media/repo-based)。

```javascript
const showcaseTemplate: Template = {
  label: 'Showcase',
  name: 'showcase',
  ui: {
    previewSrc: '/img/blocks/features.png',
  },
  fields: [
    //...
  ],
};
```

## 自定义列表标签

默认情况下，对象列表使用模板`label`或对象的`label` + `" Item"`标记每个元素。

![list UI](/img/docs/extending-tina/Extending_Tina_No_List_Props.png)

如果列表中有多个对象，可以通过`itemProps`函数属性为每个元素自定义。

* 如果使用`fields`，请在对象上设置此项
* 如果使用`templates`，请在每个模板上设置此项

![List UI with label prop](/img/docs/extending-tina/Extending_Tina_Label_List_Props.png)

## 自定义模板标识符

使用模板属性时，您的文档会使用键`_template`存储每个选择的模板。

```markdown
---
farm:
  - name: Tina
    _template: llama
  - name: Napoleon
    _template: farmer
---
```

您可以使用`templatesKey`属性自定义此项——使用给定的键代替。

## 示例

### 简单字段

```javascript
{
  label: "Testimonial",
  name: "testimonial",
  type: "object",
  fields: [
    {
      label: "作者",
      name: "author",
      type: "string"
    },
    {
      label: "引用",
      name: "quote",
      type: "string",
      ui: {
        component: "textarea"
      }
    }
  ]
}
```

### 简单字段（模板）

```javascript
{
  label: "页面块",
  name: "pageBlocks",
  type: "object",
  list: true,
  templates: [
    {
      label: "CTA",
      name: "cta",
      fields: [...]
    },
    {
      label: "推荐",
      name: "testimonial",
      fields: [...]
    }
  ]
}
```

![](/img/docs/reference/page-blocks.png)

### 简单包装字段

数字、布尔、日期时间、引用和富文本字段类型可以用作对象的唯一字段，以创建这些类型之一的列表。

```javascript
{
  label: "作者列表",
  name: "authorList",
  type: "object",
  list: true,
  fields: [
    {
      label: '作者',
      name: 'author',
      type: 'reference',
      collections: ['author'],
    },
  ]
}
```

### 嵌套对象字段

此示例同时使用模板和字段。

* `featureItem`对象存储一个对象数组`{ headline, description, buttons }`。
* 列表中的每个功能项都有一个唯一的标签，由itemProps函数设置。
* `featureBlock`数组的buttons元素也是一个对象列表，有3个选项可供选择（模板）。

```javascript
import type { Template, TinaField } from 'tinacms';

export const featuresTemplate: Template = {
  label: '功能块',
  name: 'featureBlock',
  fields: [
    {
      name: 'featureItem',
      label: '功能项',
      type: 'object',
      list: true,
      ui {
      itemProps: (item) => {
        return { label: item?.headline };
      },
    },
    fields: [
      { name: 'headline', label: '标题', type: 'string' },
      {
        name: 'description', label: '描述',
        type: 'string', ui: { component: 'textarea' }
      },
      {
        name: 'buttons',
        label: '按钮',
        list: true,
        type: 'object',
        ui: {
          visualSelector: true,
          itemProps: (item) => {
            return { label: item?.label };
          },
        },
        templates: [
          actionsButtonTemplate as Template,
          codeButtonTemplate as Template,
          modalButtonTemplate as Template
        ],
      },
    ] as TinaField[],
    },
  ],
};
```

### 带默认值的字段

当向列表中添加新项目时，它将使用这些`defaultItem`值创建。

```javascript
{
  label: "推荐",
  name: "testimonials",
  type: "object",
  list: true,
  defaultItem: {
    author: "Judith Black",
    role: "CEO",
    quote: "Lorem ipsum dol..."
  }
  fields: [
    {
      name: "author",
      type: "string"
    },
    {
      name: "role",
      type: "string"
    },
    {
      name: "quote",
      type: "string"
      ui: {
        component: "textarea"
      }
    }
  ]
}
```

### 带自定义引用键的字段

在此处设置templateKey会将我们的保存数据更改为如下所示，其中定义模板的键已设置为`"type"`。

```javascript
{
  name: "farm",
  type: "object",
  list: true,
  templateKey: "type",
  templates: [
    {
      name: "llama",
      fields: [
        {
          name: "name",
          type: "string",
        },
      ],
    },
    {
      name: "farmer",
      fields: [
        {
          name: "name",
          type: "string",
        },
      ],
    },
  ],
},
```

```markdown
---
farm:
  - name: Tina
    type: llama
  - name: Napoleon
    type: farmer
---
```
