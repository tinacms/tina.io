每个集合都是一个独特的内容结构。在TinaCMS中创建内容文档是基于每个集合进行的，以便：

* 内容结构与集合定义相匹配。
* 文档存在于该集合下。

![](/img/docs/schema/SCR-20250312-nnbi.png)

每个集合为TinaCMS编辑器定义其自己的表单字段。

这些可以在集合的`fields`或`templates`数组中指定。

可以设置`format`属性来控制在集合中创建的内容的文件类型。

### 名称

`name`定义了在GraphQL模式中使用的集合名称。

**对于每个集合应该是唯一的。**

我们建议在集合中使用单数命名（post，而不是post**s**）。

<WarningCallout
  body={<>
    此名称不能包含空格、破折号或特殊字符。
  </>}
/>

### 路径

`path`让Tina知道哪些目录包含集合的文档。

[`Format`](#format)或[`match`](#match)属性可以进一步指定包含的文档。

不同集合的文档可以共存，但如果有重叠的文档，CLI将打印警告。

默认情况下，路径是相对于CLI运行的位置，但可以通过使用[`--rootPath`](/docs/cli-overview) CLI标志来更改。

<WarningCallout
  body={<>
    在**本地模式**下，当更改**`path`**时，您需要运行`tinacms dev`。这确保生成的文件包含新路径。
  </>}
/>

### 文件格式

`Format`定义了此集合中文档的文件扩展名。设置为`"md"`、`"markdown"`、`"mdx"`、`"json"`、`"toml"`或`"yaml"`之一。

如果未指定，则使用markdown文件。

## 路径匹配

### match.include

一个[glob模式](https://en.wikipedia.org/wiki/Glob_\(programming\))，用于匹配`path`目录中的文件子集，不包括文件扩展名。使用此字段来**排除子目录**或**匹配特定文件集**。

* `?` 匹配单个字符
* `*` 匹配任意数量的任意字符（除`/`外）
* `[abc]` 匹配给定集合中的任意一个字符
* `[a-z]` 匹配一个范围。
* `{foo,bar}` 匹配给定集合中的任意一个完整单词

最终包含的文件集由`<path>/<match.include>.<format>`定义。

要获取post目录中的所有markdown文件（即`content/posts/*.md`），但不包括子目录，请使用：

```ts
{
  path: 'content/posts',
  match: {
    include: '*',
  },
  format: 'md'
}
```

要仅获取两个文件`content/config/foo.json`和`content/config/bar.json`，请使用：

```ts
{
  path: 'content/config',
  match: {
    include: '{foo,bar}',
  },
  format: 'json'
}
```

### match.exclude

这与`match.include`的工作方式相同，但会排除与模式匹配的任何文件。排除您不希望编辑的文件，或属于不同集合的文件。

最终排除的文件集由`!(<path>/<match.exclude>.<format>)`定义。

要从集合中排除所有`index.md`文件，请使用：

```ts
{
  path: 'content/posts',
  match: {
    exclude: '**/index',
  },
  format: 'md'
}
```

## 定义默认值

`defaultItem`属性定义了在**文档创建**时应该存在的数据。默认项是一个函数，接收`collection`并返回默认项。

默认项的所有属性应对应于集合中定义的字段。

```ts
defaultItem: () => {
  return {
    {{field name}}: {{value}}
  }
}
```

### 对象的默认值

[对象字段类型](/docs/reference/types/object)独立定义其默认值。更多信息请参见此处。

### 富文本的默认值

要为[富文本字段](/docs/reference/types/rich-text/)设置默认值，您必须提供一个抽象语法树（AST）。更多信息请参见此处。

## 全局集合

> 全局集合需要tinacms: 3.9.4或更高版本。在旧版本中，集合仍然可以工作，但它出现在常规集合列表中，而不是在**站点**下。

设置`ui.global: true`以使集合成为**全局**。全局集合包含一个单一的、站点范围的设置文档——例如页眉和页脚内容、导航、主题选项或SEO默认值——而不是多个文档的列表。

在编辑器中，全局集合的呈现方式与常规集合不同：

* 它在侧边栏的**站点**标题下出现一次（带有地球图标），而不是在主集合列表中。
* 打开它直接进入文档的表单——没有文档列表需要浏览，因为全局集合旨在包含单个文档。

![在TinaCMS侧边栏的站点标题下列出的全局集合（Global, SEO Defaults, Announcement Bar），每个都有一个地球图标，与上面的常规集合列表分开。](/img/docs/reference/global-collections.png)

```ts
{
  label: 'Global',
  name: 'global',
  path: 'content/global',
  format: 'json',
  ui: {
    global: true,
  },
  fields: [
    // 例如：header, footer, theme, seo …
  ],
}
```

<WarningCallout
  body={<>
    全局集合旨在包含单个文档。如果它包含零个或多个文档，编辑器将回退到常规列表视图。
  </>}
/>

## 示例

### 简单的markdown集合

默认情况下，集合中的文件将采用markdown格式。

```ts
export default defineConfig({
  //...
  schema: {
    collections: [
      {
        name: 'posts',
        label: '博客文章',
        path: 'content/posts',
        fields: [
          // 字段数组
        ],
      },
    ],
  },
});
```

### 具有受限文件管理权限的JSON集合

允许的操作属性控制用户是否可以从集合页面创建、删除和组织内容文件。

```ts
export default defineConfig({
  //...
  schema: {
    collections: [
      {
        label: '导航',
        name: 'navigation',
        path: 'content/navigation',
        ui: {
          // 不允许编辑者创建新的导航项
          allowedActions: {
            create: false,
            delete: false,
            createNestedFolder: false,
          },
        },
        format: 'json',
        fields: [
          // 字段数组
        ],
      },
    ],
  },
});
```

### 具有默认标题的Markdown集合

```js
export default defineConfig({
  // ...
  schema: {
    collections: [
      {
        label: '博客文章',
        name: 'post',
        path: 'content/posts',
        defaultItem: () => {
          return {
            // 当创建新文章时，标题字段将设置为“新文章”
            title: '新文章',
          };
        },
        fields: [
          {
            type: 'string',
            label: '标题',
            name: 'title',
          },
        ],
      },
    ],
  },
});
```
