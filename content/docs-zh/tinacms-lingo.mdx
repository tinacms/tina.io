## TinaCMS 编辑术语

随着TinaCMS的不断发展，在描述其编辑体验时使用清晰且一致的语言是很重要的。我们经常引用一组术语来解释可视化编辑的工作原理以及编辑者如何与内容互动。以下定义概述了这些术语，并展示了它们如何结合在一起：

### 管理侧边栏

列出集合的侧边栏（不要与TinaCMS编辑器混淆）。

![](/lingo/tinacms-admin-sidebar.jpeg "左侧管理侧边栏")

### 可视化编辑

整体的双窗格编辑体验，左侧为**编辑器**，右侧为实时**网站预览**。

![](/lingo/visual-editing.jpeg "左侧为编辑器，右侧为网站预览的可视化编辑")

### TinaCMS 编辑器

左侧窗格，通过结构化字段和输入来编辑内容。

![](/lingo/tinacms-editor.jpeg "左侧TinaCMS编辑器")

### 网站预览

右侧窗格，显示网站的实时视图，随着内容的编辑而更新。

### 可编辑区域

在**网站预览**中带有蓝色轮廓 🟦 的元素是可编辑的。点击任何带轮廓的元素将自动聚焦到侧边栏中对应的字段以便立即编辑。

![](/lingo/visual-highlight.jpeg "标题的视觉元素高亮")

### 字段

[字段](https://tina.io/docs/reference/fields)定义了提供给你的网站的内容结构。这些字段在你的配置中或在一个单独的`模板`中定义。

```ts
const someTemplate: Template = {
  name: 'someTemplate',
  label: '某个模板',
  fields: [
    { name: 'title', label: '标题', type: 'string' }
   ]}
```

![](</lingo/tinacms-fields.jpeg> "TinaCMS编辑器中高亮的字段")

### 富文本嵌入

在[富文本字段](/docs/reference/types/rich-text)中，我们可以选择将自定义组件作为模板嵌入。这允许你用React组件丰富基于Markdown的内容。

```
const someCollection: Collection = {
  name: 'someCollection',
  label: '某个集合',
  path: 'content/someCollection',
  format: 'mdx',
  fields: [
    {
    name: 'richBody',
    label: '丰富的正文',
    type: 'rich-text',
    templates: [
      someTemplate
      ]}
     ]
    }
```

![](</lingo/tinacms-rich-text-embed.jpeg> "富文本正文中的富文本嵌入")

### 块

[块](/docs/editing/blocks)是一种建模灵活页面构建器风格内容的方法。它们是不同组件（部分）的有序数组，如英雄图、功能、内容等。

![](/lingo/tinacms-editor.jpeg "“页面部分”下的块数组")

### 编辑工作流

[编辑工作流](/editorial-workflow)允许用户在基于Git的环境中轻松地在TinaCMS中进行更改。它为用户提供了以下能力：

* 在TinaCMS编辑器中创建分支
* 从自己的网站进行Git提交
* 在TinaCMS编辑器中创建拉取请求
