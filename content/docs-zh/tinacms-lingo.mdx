## TinaCMS 编辑术语

随着 TinaCMS 的不断发展，在描述其编辑体验时使用清晰且一致的语言非常重要。我们经常引用一组术语来解释可视化编辑的工作原理以及编辑者如何与内容互动。以下定义概述了这些术语，并展示了它们如何结合在一起：

### 管理侧边栏

列出集合的侧边栏（不要与 TinaCMS 编辑器混淆）。

![](/img/docs/lingo-admin-sidebar.png)

### 可视化编辑

整体的双窗格编辑体验，左侧是 **编辑器**，右侧是实时的 **网站预览**。

![](/img/docs/lingo-visual-editing.png)

### TinaCMS 编辑器

左侧窗格，通过结构化字段和输入来编辑内容。

![](/img/docs/lingo-tinacms-editor.png)

### 网站预览

右侧窗格，显示网站的实时视图，随着内容的编辑而更新。

### 可编辑区域

在 **网站预览** 中带有蓝色轮廓 🟦 的元素是可编辑的。点击任何带轮廓的元素将自动聚焦到侧边栏中对应的字段以便立即编辑。

![](/img/docs/lingo-editable-regions.png)

### 字段

[字段](https://tina.io/docs/reference/fields) 定义了提供给你的网站的内容结构。这些字段在你的配置中或在一个单独的 `Template` 中定义。

```ts
fields: [
  { 
    name: 'image', 
    label: '图片', 
    type: 'image' 
  },
  { 
    name: 'content', 
    label: '主要内容', 
    type: 'rich-text' 
  }
]
```

![](/img/docs/lingo-fields.png)

### 富文本嵌入

在 [富文本字段](/docs/reference/types/rich-text) 中，我们可以选择将自定义组件作为模板嵌入。这允许你使用 React 组件丰富基于 Markdown 的内容。

```ts
const document: Collection = {
  name: 'pageContent',
  label: '页面',
  path: 'content/page',
  format: 'mdx',
  fields: [
    {
      name: 'content',
      label: '主要内容',
      type: 'rich-text',
      templates: [
        youtubeEmbed
      ]
    }
  ]
}
```

![](/img/docs/lingo-rich-text-embed.png)

### 块

[块](/docs/editing/blocks) 是一种建模灵活页面构建器风格内容的方法。它们是不同组件（部分）的有序数组，例如英雄图、功能、内容等。

![](/lingo/tinacms-editor.jpeg "“页面部分”下的块数组")

### 编辑工作流

[编辑工作流](/editorial-workflow) 允许用户在基于 Git 的环境中轻松地在 TinaCMS 中进行更改。它为用户提供了以下能力：

* 在 TinaCMS 编辑器中创建分支
* 从他们自己的网站进行 Git 提交
* 在 TinaCMS 编辑器中创建拉取请求
