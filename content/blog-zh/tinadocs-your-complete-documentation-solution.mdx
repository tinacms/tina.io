在过去的几个月里，我们一直在构建一个新的文档入门模板，旨在让团队更容易使用TinaCMS创建、编辑和维护文档。这个项目现在已公开发布，名为[TinaDocs,](https://tina.io/tinadocs) 是一个专门针对基于Next.js和TinaCMS构建的完整结构文档设置的入门包。

如果想在不安装的情况下试用它 - 试试[在线演示沙盒](https://docsdemo.tina.io/admin/index.html#/~/docs)!

![](/uploads/welcome-to-tinadocs.jpeg)

## 为什么文档很重要

Markdown长期以来一直是技术文档的标准格式。因此，文档自然成为TinaCMS最强大和最常见的用例之一。Tina的编辑模型是为了利用Markdown的简单性而构建的。随着时间的推移，我们看到许多团队在TinaCMS之上构建自己的文档设置，每个团队都有自己的特定需求构建。

虽然这种灵活性很强大，但也意味着团队在从头开始构建类似的组件、导航系统和工作流程时是在重新发明轮子。我们认识到有机会将这一切整合成一个连贯的、经过实战考验的入门模板，团队可以根据自己的确切需求进行定制。

## 介绍TinaDocs

TinaDocs结合了多年的文档最佳实践，具有个性化的可定制性和丰富的功能集。它适用于开发人员和非开发人员，使您的整个团队可以轻松地贡献文档而无需接触代码。

### TinaDocs的特别之处

**无专有数据存储** - 您的内容完全存储在您的GitHub存储库中 - 版本化、可移植，并完全由您控制。没有供应商锁定，没有数据库依赖

**基于Next.js构建** - 利用Next.js的性能和开发者体验，开箱即用的服务器端渲染、静态站点生成和优化构建

**TinaCMS集成** - 通过TinaCMS直观的可视化编辑器无缝编辑内容，或直接处理Markdown文件。您的团队可以选择最适合他们的工作流程

## 丰富的功能集

TinaDocs配备了构建世界级文档所需的一切：

* **由PageFind驱动的快速搜索** - 闪电般快速的客户端搜索，离线工作且不需要后端。您的用户可以立即找到他们需要的内容
* **从OpenAPI规范生成的API文档** - 使您的[API文档](https://tina.io/tinadocs/docs/tinadocs-features/openapi-spec-docs)与您的代码保持同步。只需将TinaDocs指向您的OpenAPI规范，它会自动生成美观、互动的API文档，包含请求/响应模式和示例代码
* **用于复杂文档布局的多标签界面** - 使用标签界面组织复杂信息，非常适合跨多种语言的代码示例、不同的实现方法或分步教程
* **自定义主题以匹配您的品牌** - [六个精心设计的主题](https://tina.io/tinadocs/docs/tinadocs-features/styling)开箱即用，主题系统使您可以轻松自定义颜色、排版和样式，以完美匹配您的品牌标识
* **大量标准化的文档组件** - 从带语法高亮的代码块到提示框、手风琴、卡片网格等。您需要的每个组件都包含在内，以创建引人入胜、易于阅读的文档。在[TinaDocs文档](https://tina.io/tinadocs/docs/tinadocs-features/feature-list)中找到完整列表
* **SEO优化** - [内置的SEO功能](https://tina.io/tinadocs/docs/tinadocs-features/search-engine-optimization-seo)确保您的文档在搜索引擎中排名良好。元标签、结构化数据和站点地图生成都自动处理

## 入门指南

### CLI

准备好试试TinaDocs了吗？入门很简单：

```bash
# 克隆存储库
git clone https://github.com/tinacms/tina-docs.git

# 安装依赖
cd tina-docs
pnpm install

# 启动开发服务器
pnpm dev
```

### TinaCloud快速入门

前往[TinaCloud快速入门,](https://app.tina.io/quickstart) 登录并选择TinaDocs入门选项

![](/uploads/tina-cloud-quickstart.jpeg)

## 由社区构建，为社区服务

TinaDocs是开源的，可以在[GitHub](https://github.com/tinacms/tina-docs)上找到。我们在构建时考虑了可扩展性，每个组件、每个主题、每个功能都可以根据您的特定需求进行定制。无论您是在记录库、API、内部流程还是用户指南，TinaDocs都提供了一个坚实的基础，您可以在此基础上构建。
