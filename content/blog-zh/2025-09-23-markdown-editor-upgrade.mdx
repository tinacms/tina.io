在我们最新发布的Markdown编辑器中，TinaCMS带来了**五大改进**，旨在让编辑更快、更简洁、更安全。从代码块和图表到内容结构和依赖更新，此次升级旨在优化写作流程，同时降低风险。

## 为什么这些变化很重要

Markdown是TinaCMS用户创建和维护内容的核心。当编辑感觉笨拙，或当错误和漏洞出现时，它会拖慢从开发人员到内容团队的每一个人。\
此次发布通过UI改进、更好的插件处理和更严格的安全性直接解决了这些问题。

## 新功能

### 1. 更智能的代码块与可搜索的下拉菜单

代码块是最受欢迎的改进请求。在以前的版本中，空块有时会显示“未定义”，下拉菜单可能会破坏UI。

现在代码块是：

* **可搜索的** — 从下拉菜单中选择你的语言。
* **稳定的** — 不再有“未定义”的故障。
* **更简洁易用** — 改进的UI带来更流畅的编辑体验。

### 2. 将Mermaid图表集成到代码块中

Mermaid之前是一个独立的组件。现在不再是这样。

* 直接从代码块下拉菜单中选择`mermaid`。
* 在线编写图表。
* 使用预览按钮即时检查图表。

不再需要来回切换。

### 3. 水平分割线插件实现更清晰的分隔

对于长内容，结构很重要。新的水平分割线插件让你可以用一条线插入清晰的章节分隔，使文本更易于阅读。

### 4. 防止不支持的Markdown和插件

以前，不支持的插件或Markdown可能会被添加到代码块中，有时会破坏格式。\
现在，不支持的选项默认被禁用——防止错误并保持内容稳定。

### 5. 安全性和依赖更新

保持依赖的最新状态至关重要。在此版本中，我们：

* 更新了TinaCMS依赖到最新版本。
* 对各个包的安全漏洞进行了基准测试。
* 通过更新一个关键依赖，将易受攻击的包从**39个减少到26个**。

这意味着你的项目更安全，更易于维护。

## 升级路径

升级到最新版本很简单。更新你的依赖，你将立即获得新的编辑器体验：

```bash
npm install tinacms@latest
# 或者
yarn add tinacms@latest
```

查看[文档](https://tina.io/docs)以获取有关使用代码块、Mermaid图表和新插件的详细指南。

## 展望未来

此次升级专注于**更好的用户体验和更强的安全性**，但这仅仅是个开始。我们致力于在未来的版本中进一步完善编辑器，提供更多的插件支持、改进的预览和更快的性能。
